## 2. node --check JS 语法验证

在执行浏览器验收之前，先对所有 JavaScript 文件进行语法检查：

```bash
# 检查所有 JS 文件
find . -name "*.js" -not -path "*/node_modules/*" -exec node --check {} \;
```

若所有文件无输出，表示语法检查通过。

## 5. 搜索验收

- **目的**：验证搜索框的关键词过滤功能对所有图表均生效
- **方法**：输入关键词后，检查地图气泡、排行图、里程图、强度图是否只显示匹配城市
- **测试用例**：
  - 输入"厦门"：仅显示厦门相关数据
  - 输入"广"：显示所有名称包含"广"的城市
  - 清空搜索框：恢复全部城市数据
- **结果**：全部通过

---

## 6. 指标切换验收

- **目的**：验证指标切换后 tooltip、单位标签、地图气泡大小均正确更新
- **测试指标**：客流量（万人次）、营业里程（km）、客流强度（万人次/km）
- **验证点**：
  - tooltip 显示对应指标的数值和单位
  - 地图气泡大小反映对应指标的数值
  - 排行图表标题和数值对应所选指标
- **结果**：全部通过

## 7. Top N 验收

- **目的**：验证 Top N 下拉选择对不同排行图表的影响
- **测试选项**：客流排行、里程排行、强度排行
- **验证点**：

  - 排名顺序与数据一致
  - 切换后图表无残留旧数据
- **结果**：全部通过

## 8. 详情面板验收

## 9. 移动端验收

- **目的**：验证在移动端视口（375px 宽度）下页面无水平滚动
- **方法**：将 puppeteer 视口设置为 375x812（iPhone X 尺寸），检查 `document.documentElement.scrollWidth <= 375`
- **结果**：通过，无水平滚动条

## 10. 后续验收要求

> **每次代码变更后必须执行以下验收流程：**

1. **JS 语法检查**：`node --check` 对所有修改过的 `.js` 文件进行检查
2. **浏览器验收**：运行完整的浏览器测试脚本（T01-T11），确保 16/16 PASS
3. **回归验证**：若变更涉及数据，额外检查数据完整性（城市数量、数值合理性）
4. **文档更新**：若变更涉及功能或数据，同步更新相关文档
5. **提交前确认**：`git status` 确认无遗漏文件，`git diff` 确认无意外修改
