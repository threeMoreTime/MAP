## 数据问题

### 1. 14 个城市无 MetroDB 数据

- **严重程度**：中
- **描述**：以下 14 个城市在 MetroDB 数据源中无客流数据记录：
  - 具体城市列表（根据实际数据确认）
- **影响**：这些城市在 Dashboard 中显示"暂无数据"，无法进行客流分析
- **计划**：Phase 2 中排查数据源覆盖情况，考虑补充其他数据源

### 2. 8 个城市日客流数据为零

- **严重程度**：中
- **描述**：以下 8 个城市虽有数据记录，但 `daily_ridership_wan`（日客流量/万人次）值为 0：
  - 具体城市列表（根据实际数据确认）
- **影响**：排行榜和指标计算中这些城市显示异常（客流强度为 0 或无穷大）
- **计划**：Phase 2 中进行数据清洗，标记或过滤异常值

## 架构问题

### 3. DATA 对象嵌入在 dashboard.html 中

- **严重程度**：中
- **描述**：所有城市数据以 JavaScript 对象（`DATA`）的形式直接硬编码在 `dashboard.html` 文件内
- **影响**：
  - 数据更新需要修改 HTML 文件
  - 文件体积较大
  - 无法独立缓存数据和页面
  - 多页面无法复用同一份数据
- **计划**：Phase 4 将数据外置为独立 JSON 文件

### 4. ECharts CDN 依赖

- **严重程度**：低
- **描述**：Dashboard 依赖 CDN 加载 ECharts 库（`https://cdn.jsdelivr.net/npm/echarts@5/dist/echarts.min.js`）
- **影响**：
  - 离线环境无法使用
  - CDN 服务不可用时页面无法渲染图表
  - CDN 版本变更可能导致兼容性问题
- **计划**：后续考虑将 ECharts 本地化，或使用 Subresource Integrity (SRI) 校验

## 工程化问题

### 5. 无 CI/CD 流水线

- **严重程度**：中
- **描述**：项目未配置 GitHub Actions 或其他 CI/CD 工具
- **影响**：
  - 代码提交无法自动验证
  - 数据更新需手动执行完整流程
  - 无法自动发布
- **计划**：Phase 3 配置自动化测试，Phase 6 配置自动数据采集

### 6. 无 GitHub Pages 发布

- **严重程度**：低
- **描述**：项目尚未配置 GitHub Pages，无法通过 URL 直接访问 Dashboard
- **影响**：查看 Dashboard 需要本地克隆并启动静态服务
- **计划**：Phase 5 配置 GitHub Pages

### 7. 数据非实时

- **严重程度**：低
- **描述**：Dashboard 中的数据为静态快照，非实时数据
- **影响**：数据可能存在滞后，无法反映最新客流状况
- **计划**：Phase 6 通过定时任务实现定期数据更新

## 资源问题

### 8. 本地 GeoJSON 文件体积较大（582KB）

- **严重程度**：低
- **描述**：仓库中包含一个约 582KB 的中国地图 GeoJSON 文件
- **影响**：
  - 增加仓库克隆时间
  - 占用存储空间
- **计划**：后续考虑使用 CDN 加载地图数据，或对 GeoJSON 进行简化/压缩
