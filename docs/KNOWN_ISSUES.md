# 已知问题清单

> 仓库：threeMoreTime/MAP

## 数据问题

### 1. 16 个城市无 MetroDB 数据

- **严重程度**：中
- **描述**：以下 16 个城市在 MetroDB 数据源中无客流数据记录：
  - 福州、宁波、温州、济南、洛阳、徐州、乌鲁木齐、香港、澳门、高雄、台中、台北、金华、台州、佛山、绍兴
- **影响**：这些城市在 Dashboard 中显示"暂无数据"，无法进行客流分析
- **计划**：Phase 2 中排查数据源覆盖情况，考虑补充其他数据源

### 2. 8 个城市日客流数据为零

- **严重程度**：中
- **描述**：杭州、青岛、大连、呼和浩特、太原、芜湖、无锡、南通共 8 个城市虽有数据记录，但 `daily_ridership_wan`（日客流量/万人次）值为 0
- **影响**：排行榜和指标计算中这些城市显示异常（客流强度为 0 或无穷大）
- **计划**：已做数据清洗标记，Dashboard 中显示"暂无数据"，不纳入统计汇总

## 架构问题

### 3. DATA 对象嵌入在 dashboard.html 中

- **严重程度**：中
- **描述**：所有城市数据以 JavaScript 对象（`DATA`）的形式直接硬编码在 `dashboard.html` 文件内
- **影响**：
  - 数据更新需要修改 HTML 文件
  - 文件体积较大
  - 无法独立缓存数据和页面
  - 多页面无法复用同一份数据
- **计划**：Phase 4 将数据外置为独立 JSON 文件

### 4. ECharts CDN 依赖

- **严重程度**：低
- **描述**：Dashboard 依赖 CDN 加载 ECharts 库（`https://cdn.jsdelivr.net/npm/echarts@5/dist/echarts.min.js`）
- **影响**：
  - 离线环境无法使用
  - CDN 服务不可用时页面无法渲染图表
  - CDN 版本变更可能导致兼容性问题
- **计划**：后续考虑将 ECharts 本地化，或使用 Subresource Integrity (SRI) 校验

## 工程化问题

### 5. 无 CI/CD 流水线（已解决，远端验证通过）

- **严重程度**：低
- **描述**：自动化持续集成流水线 `.github/workflows/ci.yml` 已设计、配置并上线验证通过，涵盖基线和 React 构建、UI 测试隔离 Job，远端跑通，实现 100% 自动化质量控制。

### 6. 无 GitHub Pages 发布（已解决，远端验证通过）

- **严重程度**：低
- **描述**：自动化部署工作流 `.github/workflows/pages.yml` 已支持自动 CD，配置了基于 master push 且 CI 成功后的自动部署逻辑（同时保留手动触发作为兜底），并在部署完成后自动运行线上 Pages Smoke Test 冒烟测试，远端验证已全部通过。

### 7. 数据非实时

- **严重程度**：低
- **描述**：Dashboard 中的数据为静态快照，非实时数据
- **影响**：数据可能存在滞后，无法反映最新客流状况
- **计划**：Phase 6 通过定时任务实现定期数据更新

## 资源问题

- **严重程度**：低
